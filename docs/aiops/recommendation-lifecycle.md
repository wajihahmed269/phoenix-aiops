# Recommendation Lifecycle

## Purpose

Define the operator-visible lifecycle for Phoenix-Ops recommendations before any execution layer exists.

## States

- `proposed`: generated by the polling pipeline and stored for review
- `acknowledged`: seen by an operator and retained for follow-up
- `suppressed`: intentionally muted because it is a known limitation or currently not actionable
- `rejected`: explicitly dismissed by an operator
- `expired`: aged out by future retention logic

## Current Flow

```text
collect -> normalize -> analyze -> policy -> dedupe -> persist -> review
```

## Current Review Actions

- list recommendations
- fetch one recommendation
- acknowledge a recommendation
- suppress a recommendation

## Not In Scope Yet

- approve for execution
- execute remediation
- auto-close after mutation
- bidirectional sync with Argo CD or Grafana
