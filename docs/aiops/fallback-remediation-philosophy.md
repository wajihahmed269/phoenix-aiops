# Fallback Remediation Philosophy

Fallback remediation in Phoenix-Ops is a future survivability feature, not a shortcut to autonomous AI execution.

This phase prepares the prerequisites:

- cataloged actions
- approval state
- guardrail evaluation
- rollback snapshot capture
- execution audit history
- verification structure
- escalation stage tracking

K8sGPT remains advisory. It can improve diagnostic readability, but it does not select or authorize remediation.

Simulation-first matters because it lets the platform prove orchestration behavior, audit persistence, and verification sequencing before real mutation is allowed.
